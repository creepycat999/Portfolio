Responsive Redesign

The goal of this assignment was to practice the workflow of redesigning a simple website. I took away the skills necessary to analyze and identify flaws in an existing interface, create low-fidelity and high-fidelity prototypes for various screen sizes, and build a responsive website based on those prototypes.

Part 1: Identifying Usability Problems

In this part, I chose one page of a movie theater website to analyze based on usability and accessibility standards. Then redesigned and build the redesigned version of this webpage.


This website has a lot of character which I really appreciate and want to preserve. However, it was also perfect for a responsive redesign since the website doesn't even take up an entire browser screen. I found that it was made up of mostly images and not css or html, so it was not very adaptive to different sizes.

Usability



Effeciency: This UI is not very effecient, as there is no clear heirarchy of functionality. Buttons exist on non-linear planes and do not look like eachother.


Learnability: It is hard to know where to click in the website, since buttons are not explicit. I expect a lot of trail an error on first use to try and navigate around the website to see which asepcts are photos and which are interactive.


Memorability: Memorbility is easier once you have used the website a lot, however it is not like any other website and errors can absolutely appear. Once again, buttons like "show times" "coming soon" and "now playing" are hidden in the page.
